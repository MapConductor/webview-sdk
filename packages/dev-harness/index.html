Simulated native side

Plays the role of Android/iOS in a plain browser tab — sends the same command/event envelopes a real WebView bridge would, against the reference runtime in the iframe. For fast JS-side iteration without a native build; it can't reproduce WebView-engine-specific rendering bugs (e.g. the real-device height: 100% bug — see android-for-webview-leaflet's README), so this passing isn't a substitute for on-device verification.
